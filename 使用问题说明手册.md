## 1. 软件无法启动

### ❓ 双击图标后无任何反应
**原因**：系统缺少 Visual C++ 运行库。  
**解决方案**：下载并安装 [Microsoft Visual C++ Redistributable](https://learn.microsoft.com/zh-cn/cpp/windows/latest-supported-vc-redist) (x64 版本)，安装完成后重新运行软件。

### ❓ 弹出"找不到 DLL"错误
**原因**：串口驱动库 (`libserialport`) 未能加载。  
**解决方案**：
1. 确保软件所在目录下的所有 `.dll` 文件完整，不要将 `.exe` 单独拷贝到其他目录运行。
2. 以完整压缩包形式分发和部署软件。

### ❓ 启动时闪退，什么提示都没有
**原因**：数据库初始化失败（通常是首次运行时文档目录写入权限不足）。  
**解决方案**：
1. 右键点击软件图标，选择"以管理员身份运行"，让软件完成一次初始化。
2. 后续正常运行无需管理员权限。

## 2. 串口连接问题

### ❓ 串口列表为空，找不到 COM 口
**原因**：RS485 转 USB 驱动未安装，或设备未被系统识别。  
**解决方案**：
1. 打开 Windows 设备管理器（`Win + X` → 设备管理器）。
2. 检查"端口 (COM 和 LPT)"下是否有对应设备。若有黄色感叹号，需重新安装对应的 USB 转串口驱动（常见型号：CH340、CP210x、FTDI）。
3. 安装驱动后重新插拔设备，再刷新软件中的串口列表。

### ❓ 串口已选择，点击连接后失败或弹出错误
**可能原因及解决方案**：
- **COM 口被其他程序占用**：关闭其他可能占用串口的程序（如串口调试助手），再重试。
- **波特率设置不匹配**：确保软件中的波特率与电机驱动器/采集模块拨码设置一致（常用：9600 / 19200 / 38400）。
- **选错 COM 口**：COM_A 对应电机控制总线，COM_B 对应电流采集模块总线，请勿接反。

### ❓ 连接成功但电机无响应
**解决方案**：
1. 检查 RS485 接线（A/B 线不能接反）。
2. 检查终端电阻（总线末端需接 120Ω 电阻）。
3. 核对电机驱动器拨码设置的设备地址与软件中通道 ID（CH-01 对应地址 1）是否一致。

## 3. 电机无法启动

### ❓ 点击播放键后卡片没有任何变化
**原因排查顺序**：

| 检查项 | 说明 |
|--------|------|
| 串口是否已连接 | 顶部"串口配置"按钮旁的图标为绿色才表示已连接 |
| 是否已扫码绑定 | 卡片上的"待扫码"状态表示尚未绑定产品二维码，**未绑定不允许启动** |
| 是否已配置工况 | 卡片显示"未配置"时，需在配置页新建工况再分配给对应通道 |
| 是否处于报警锁定 | 卡片红色高亮需先点击复位键才能再次启动 |

### ❓ 提示"工况未配置"无法启动
**解决方案**：
1. 切换至左侧"参数配置"页面，点击"新建工况"创建测试方案。
2. 回到主界面，在对应电机所在行的"配置全组方案"下拉菜单中选择刚创建的方案。

## 4. 电机启动后立即报警停机

### ❓ 启动后不到1秒就变红，显示"报警停机"
**可能原因**：
1. **电流下限设置过高**：当实际空载电流低于设定下限时会触发报警。请检查工况模板中"报警下限电流"设置是否合理（空载通常 < 0.5A）。
2. **电流上限设置过低**：启动浪涌电流超过上限。建议上限值设为额定电流的 1.5~2 倍。
3. **采集模块 COM_B 连接异常**：串口 B 未连接或故障时，电流读取返回 `null`，不会触发报警，但如果读取到异常值则可能误触。

### ❓ 运行几百次后才偶发报警
**可能原因**：机械磨损或发热导致实际电流升高，属于正常监测触发。  
**处理方式**：
1. 在历史查询页搜索该产品二维码，查看电流曲线趋势。
2. 记录触发时的循环次数和电流值，判断是渐进式升高还是突变。

### ❓ 串口通信中断导致误报警
**说明**：系统已内置 **3次重试机制**（间隔200ms），连续3次均失败才会触发报警。如果仍频繁误报，建议：
- 检查 485 总线的线缆质量和接地屏蔽情况。
- 降低波特率（从 19200 降到 9600）以提高通信稳定性。

## 5. 扫码枪录入问题

### ❓ 扫完码后数字接在了上一次的码后面
**说明**：此问题已在 v1.2.0 修复。请确认您使用的是最新版本。  
**工作机制**：点击/聚焦扫码框时，软件会自动清空输入框中的旧内容，直接扫新码即可。

### ❓ 扫完码后，旁边显示的"码值绑定"没有更新
**原因**：扫码枪发送的字符串末尾带有换行符 `\n` 或回车 `\r`，导致绑定的值含有特殊字符。  
**解决方案**：在扫码枪的配置软件中，设置"后缀"为"无"或取消自动添加回车/换行，再重新扫码。

### ❓ 想为一整组5台电机绑定同一个批次号
**说明**：目前每台电机卡片独立绑定二维码（因为每台对应一个独立产品）。若需批次追溯，建议统一编码规则，例如使用格式 `批次号-序号`：`BATCH001-01`、`BATCH001-02`...，在历史页面用批次号前缀模糊匹配查询。

## 6. 历史数据查询异常

### ❓ 扫码查询后提示"未找到该码值对应的数据"
**排查步骤**：
1. 确认该产品已经至少完成过一次测试运行（未运行过的产品没有数据）。
2. 核对扫码枪是否扫入了正确的码——扫码框不区分大小写，但反斜杠、空格等特殊字符需完全一致。
3. 打开数据库文件（路径见下文），执行 `SELECT * FROM current_logs LIMIT 10` 确认数据库中确实有数据。

### ❓ 查询结果中看不到报警记录，只有正常数据
**原因**：如果该产品测试过程中从未触发过报警，报警表中就没有对应记录，这是正常现象。

### ❓ 表格中的时间显示格式异常
**说明**：时间存储格式为 ISO 8601（`2026-03-16T09:30:00.000`），界面已自动截取为 `2026-03-16 09:30:00` 格式显示。如看到原始格式，属于显示层逻辑问题，不影响数据本身。

## 7. 数据库与数据管理

### 📂 数据库文件位置
```
C:\Users\<您的用户名>\Documents\MotorControl\motor_control.db
```
快速跳转：在文件资源管理器地址栏输入 `%USERPROFILE%\Documents\MotorControl\` 后回车。

### ❓ 如何备份数据
直接复制 `motor_control.db` 文件到其他位置即可完成备份。建议定期（每周/每月）手动复制至 U 盘或网络共享路径。

### ❓ 如何将数据导出为 Excel
1. 下载 [DB Browser for SQLite](https://sqlitebrowser.org/)（免费开源）。
2. 点击"打开数据库"，选择 `motor_control.db`。
3. 进入"执行 SQL"选项卡，输入查询语句后点击运行。
4. 点击结果区右键 → "导出为 CSV"，再用 Excel 打开即可。

常用导出 SQL：
```sql
-- 导出某产品全部电流历史
SELECT timestamp, loop_count, read_current 
FROM current_logs 
WHERE qr_code = '在此替换为产品码值' 
ORDER BY timestamp ASC;

-- 导出所有报警记录
SELECT timestamp, qr_code, motor_id, trip_current, limit_value 
FROM alarm_logs 
ORDER BY timestamp DESC;
```

### ❓ 数据库文件越来越大，如何清理历史数据
使用 DB Browser 打开数据库后执行以下 SQL（**请先备份！**）：
```sql
-- 清理 90 天前的电流流水数据
DELETE FROM current_logs 
WHERE timestamp < datetime('now', '-90 days');

-- 执行后运行此命令回收空间
VACUUM;
```

## 8. 报警复位相关

### ❓ 点击"全部停止"后报警的卡片红色消失了
**说明**：此问题已在 v1.2.0 修复。新版本中"停止"操作不再清除报警标志，只有**点击卡片右上角的橙色复位键**才能消除报警标识。

### ❓ 复位后能否继续之前的测试？
**可以。** 复位仅清除报警状态，`currentLoop`（当前循环圈数）**不会归零**。重新绑定扫码后点击启动，会从上次中断的循环数继续运行，直到达到目标总圈数。若希望从头开始（归零重测），需等循环数达到目标值后再次点击启动时系统会自动重置计数。

### ❓ 复位后再次启动，又立刻报警
**原因**：硬件故障尚未排除，或电流阈值设置不合理。  
**排查思路**：
1. 手动转动电机轴，判断是否有机械卡死。
2. 检查驱动器是否过热。
3. 在历史查询页查看报警前几次的电流趋势，判断是突变还是渐变到限值。

## 9. 界面显示异常

### ❓ 卡片显示"正转..."但电机已经停止了
**说明**：此状态残留问题已在 v1.2.0 修复。若在旧版本中出现，可点击"全部停止"让状态强制归位，或重启软件。

### ❓ 窗口无法拖拽移动
**提示**：无边框窗口需通过顶部**蓝色渐变标题栏区域**（"多路电机群控与数据采集系统"标题文字区域）拖拽移动，卡片区域无法拖拽。

### ❓ 右上角没有最小化/最大化/关闭按钮
**原因**：可能是在极小分辨率下按钮被裁剪。  
**解决方案**：尝试调整窗口大小或将显示器分辨率提高到 1024×768 以上（软件最低窗口尺寸限制为 1024×768）。

### ❓ 电机卡片文字显示乱码
**原因**：系统字体缺少中文 CJK 字符集。  
**解决方案**：在 Windows 设置 → 时间和语言 → 语言 → 中文简体中勾选安装"字体"可选功能。

## 10. 工况模板管理

### ❓ 新建工况后，主界面的"配置全组方案"下拉列表没有更新
**解决方案**：点击主界面顶部的**"刷新模板"按钮**，重新从数据库加载最新的模板列表。

### ❓ 误删了工况模板，能否恢复？
**新版本（v1.2.0+）**：删除前会弹出确认弹窗，可取消。  
**如果已经删除**：若之前有数据库备份，可从备份文件的 `work_mode_templates` 表中找到记录，手动重建。工况参数（步骤、循环次数、电流阈值）均储存在 `steps_json` 字段中。

### ❓ 如何配置一个"正转5秒 → 停1秒 → 反转5秒 → 停1秒"的循环工况？
**步骤**：
1. 在"工况配置"页点击"新建工况"。
2. 依次添加动作步骤：`正转 / 5秒` → `停止 / 1秒` → `反转 / 5秒` → `停止 / 1秒`。
3. 设置目标循环次数（如 1000 次）、电流采集间隔（如每 10 次采1次）、电流上下限。
4. 保存后在主界面分配给对应通道组。

## 📞 其他问题

如遇到本手册未覆盖的问题，请收集以下信息后联系开发人员：

1. **问题描述**：发生了什么、触发条件、频率。
2. **软件版本**：在 README 中查看版本号。
3. **错误截图**：如有弹窗报错请截图保存。
4. **日志信息**：若在开发调试模式运行，收集 `flutter run` 控制台中 `[SerialManager]` 或 `[MotorState]` 开头的日志行。
5. **数据库文件**：将 `%USERPROFILE%\Documents\MotorControl\motor_control.db` 文件打包提供（含完整历史数据以便复现问题）。
